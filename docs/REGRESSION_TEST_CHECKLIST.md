# 全系统回归测试清单

适用范围：学生端、教师端 / 管理端、权限隔离、响应式布局、构建检查。

测试前准备：

- 使用 `.env.local` 中的测试账号登录，不要把账号密码写入源码、文档、截图或提交记录。
- 建议先执行 `npm.cmd run lint`、`npm.cmd run build`，再启动 `npm.cmd run dev` 做人工回归。
- 涉及删除、审核、保存答案解析等会改变数据的操作时，优先使用专门测试数据。

## 复习计划生成规则

- 学生选择题型并点击“保存错题”后，错题可以立即进入复习计划。
- 学生不确定题型并点击“提交教师审核”后，错题保持 pending，暂不生成复习计划。
- 教师在 `/teacher/review-mistakes` 确认三级题型后，错题状态变为 `teacher_confirmed`，系统才为该错题生成复习任务。
- 教师后续在答案解析中心补充答案解析，不应作为生成复习计划的前置条件；它影响学生查看答案解析的完整性。
