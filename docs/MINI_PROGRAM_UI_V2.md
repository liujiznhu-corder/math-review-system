# 1. 设计目标

微信小程序学生端的核心目标是让学生每天快速完成“看任务、做复盘、补错题、查答案”的闭环。

小程序要解决：

- 学生每天快速知道今天该做什么。
- 快速录入错题。
- 完成今日复习。
- 完成薄弱巩固。
- 完成专项训练。
- 查看答案解析。

设计关键词：

- 极简
- 卡片式
- 学习感
- 数学风
- 手机优先
- 操作路径短

设计原则：

- 首页聚焦“下一步该做什么”，不堆历史日志。
- 做题页一次只展示当前题，不使用长列表上下滑动看题。
- 所有核心操作控制在 1 到 2 次点击内完成。
- 教师端能力不进入小程序，教师继续使用 Web 后台。
- 不收集手机号、邮箱、微信号、真实姓名、头像、昵称。

# 2. 全局 UI 风格

## 主色建议

主色建议使用沉稳绿色：

- 主色：`#2F5D50`
- 用途：主要按钮、关键进度、选中状态、成功反馈。

绿色表达稳定、持续学习、复盘积累，适合备考类产品。

## 辅助色建议

- 警示色：`#B85C38`
  - 用于未掌握、待处理、需要注意的状态。
- 成功色：`#2F7D5A`
  - 用于已掌握、已完成、绑定成功。
- 中性色：`#6B7280`
  - 用于说明文字、次级信息。
- 信息色：`#3B6EA8`
  - 用于链接、提示、可查看内容。

## 背景色建议

- 页面背景：`#F7F5EF`
- 卡片背景：`#FFFFFF`
- 次级块背景：`#F1EFE8`

背景不使用强渐变，不做复杂装饰，保持专注和安静。

## 卡片圆角

- 普通卡片：`8px`
- 小标签 / 小按钮：`6px`
- 弹层：`12px`

卡片圆角保持克制，不做过度柔和的娱乐化视觉。

## 阴影

阴影应轻量：

- 普通卡片：浅色边框 + 很弱阴影。
- 重点卡片：可增加轻微阴影，但不做悬浮夸张效果。
- 小程序端以边框分层为主，阴影为辅。

## 字体层级

- 页面标题：20-22px，半粗体。
- 模块标题：16-18px，半粗体。
- 正文题目：15-16px，行高充足。
- 辅助说明：13-14px。
- 数字强调：28-48px，根据场景放大。

数学题区域应优先保证可读性，公式上下留白要足够。

## 按钮样式

主要按钮：

- 高度不低于 44px。
- 宽度在手机端优先撑满容器。
- 使用主色背景和白色文字。

次要按钮：

- 白底、浅边框、深色文字。
- 用于返回、查看答案、取消等操作。

警示按钮：

- 使用浅警示背景或警示色边框。
- 用于“未掌握”“仍需巩固”等反馈。

底部固定操作区：

- 用于录题保存、提交审核、复习反馈、训练反馈。
- 避免学生滚动到底部找不到操作。

## 标签样式

标签用于：

- 题型
- 分类状态
- 来源标签
- 掌握状态

建议：

- 圆角 6px。
- 字号 12-13px。
- 背景浅色，文字颜色明确。
- 不使用强饱和大面积色块。

## 空状态样式

空状态使用轻量卡片：

- 简短标题。
- 一句解释。
- 一个下一步行动按钮。

示例：

> 还没有错题  
> 录入第一道错题后，系统会开始为你生成复习任务。

按钮：

- 去录入错题

## Loading 样式

加载状态保持安静：

- 页面级加载：骨架屏或居中 loading。
- 卡片级加载：浅色骨架条。
- 按钮提交中：按钮禁用，文案变为“处理中...”。

避免全屏强阻断，除非是首次登录和权限校验。

## 错误提示样式

错误提示使用浅警示背景：

- 标题：发生错误
- 内容：给出可理解原因
- 操作：重试 / 返回首页 / 联系老师

权限和到期类错误使用专门文案：

- 使用权限已到期，请联系老师续期。
- 当前账号暂不可用，请联系老师。

# 3. TabBar 设计

底部 TabBar 设计为 5 个入口：

1. 首页
2. 复习
3. 巩固
4. 错题
5. 我的

## 首页

定位：今日学习总览与快捷行动入口。

首页回答学生的问题：

- 今天离考试还有多久？
- 今天还有什么任务？
- 我现在应该先做什么？

## 复习

定位：完成到期错题复习任务。

复习页只处理今日复习，不承载错题库管理和训练入口。

## 巩固

定位：做额外练习。

巩固 Tab 内包含：

- 今日薄弱巩固
- 专项训练

专项训练不单独放 TabBar。

## 错题

定位：查看个人错题库和答案解析。

错题页使用卡片列表，不做复杂表格。

## 我的

定位：查看使用状态、学习摘要和系统说明。

不显示手机号、邮箱、微信号、头像、昵称、真实姓名。

# 4. 页面一：首页

首页模块顺序固定如下。

## 1. 考试倒计时大卡片

内容：

- 江苏专转本数学考试倒计时
- 距离 2027年3月21日
- 大号剩余天数
- 文案：今天多复盘一道错题，考场上就少一个失分点。

设计：

- 首页第一屏顶部展示。
- 大号数字居中。
- 卡片留白充足。
- 不放复杂图表。

## 2. 今日任务卡片

包含三个任务入口：

- 今日待复习
- 薄弱巩固
- 专项训练

每个任务卡片展示：

- 数字或状态
- 简短说明
- 点击跳转对应页面

建议布局：

- 手机端纵向卡片流。
- 可使用 2 列小卡片，但不强制横向排满。

## 3. 快捷入口

快捷入口：

- 录入错题
- 今日复习
- 薄弱巩固
- 专项训练

录入错题不放 TabBar，但首页必须有明显入口。

## 4. 薄弱题型 Top5

展示：

- 题型名称
- 掌握度
- 去巩固

设计：

- 使用紧凑卡片列表。
- 掌握度低的题型靠前。
- “去巩固”作为次级按钮或链接。

## 5. 知识点掌握度

展示：

- 一级分类名称
- 掌握度百分比
- 进度条

要求：

- 使用进度条。
- 不使用复杂图表。
- 不使用雷达图、折线图、热力图。

## 6. 最近 7 天学习总结

展示四项累计数据：

- 新增错题
- 完成复习
- 专项训练
- 薄弱巩固

建议：

- 2x2 小卡片。
- 每张卡片展示数字、指标名称、简短说明。

首页不放：

- 最近复习记录。
- 复杂学习报告。
- 长列表历史日志。

首页聚焦：

- 今天该做什么。
- 哪些题型最需要处理。
- 最近一周有没有持续学习。

# 5. 页面二：录入错题

入口：从首页“录入错题”按钮进入。

## 页面结构

### 1. AI 录题助手卡片

说明推荐流程：

```text
拍照 -> 外部 AI -> 复制提示词 -> 生成 LaTeX -> 粘贴回来
```

按钮：

- 打开 AI 录题助手

说明：

- 系统不实现 OCR。
- 系统不上传图片。
- 系统不调用 AI API。
- 学生自行使用豆包 / DeepSeek / ChatGPT 等外部工具。

### 2. 题型模板选择

题型模板：

- 单选题
- 填空题
- 计算题

交互：

- 使用分段控件或三张小卡片。
- 选中后展示对应提示词模板。

### 3. 提示词模板弹层

弹层内容：

- 模板标题
- 使用说明
- 提示词内容
- 一键复制

弹层交互：

- 底部抽屉或居中弹窗。
- 手机端优先使用底部抽屉。
- 复制成功后显示轻提示：“提示词已复制”。

### 4. LaTeX 输入框

要求：

- 只保留 LaTeX 输入。
- 不提供普通文本输入。
- 输入框使用等宽字体。
- 支持多行。
- placeholder 给出示例。

### 5. 实时预览

预览支持：

- `\blankbox`
- `\fourchoices`
- `\_\_\_`

要求：

- 长公式横向滚动。
- 渲染失败时显示原始文本和错误提示。
- 预览卡片不阻断学生继续编辑。

### 6. 题型推荐

包含：

- Top3 推荐
- 匹配分数
- 匹配理由
- 手动选择题型
- 不确定则提交教师审核

推荐卡片：

- 显示一级 / 二级 / 三级题型。
- 选中态明确。

### 7. 底部固定操作按钮

按钮：

- 保存错题
- 提交教师审核

要求：

- 手机端固定底部操作区。
- 按钮高度不低于 44px。
- 未输入 LaTeX 时按钮禁用。
- 保存错题必须已选择题型。
- 提交教师审核不要求选择题型。

# 6. 页面三：复习

复习页采用“题号导航 + 当前题详情”的任务模式。

不要使用长列表上下滑动看题。

## 1. 顶部统计

展示：

- 今日复习
- 当前进度，例如 `2/5`
- 今日已完成数量

## 2. 题号导航

展示：

- 第1题
- 第2题
- 第3题
- ...

状态：

- 已掌握
- 未掌握
- 未完成

交互：

- 支持横向滚动。
- 学生可以自由切换题目。
- 当前题号高亮。

## 3. 当前题卡片

展示：

- 题目 LaTeX
- 所属题型
- 复习轮次

题目区域：

- 支持横向滚动。
- 公式可读优先。

## 4. 查看答案

交互：

- 点击后展开答案解析。
- 或进入答案解析页。

建议 V1：

- 优先当前页展开，减少跳转。
- 如果跳转答案页，必须携带 `returnUrl`。

## 5. 操作按钮

按钮：

- 已掌握
- 未掌握

交互：

- 点击后保存反馈。
- 当前题状态立即变化。
- 自动跳到下一道未完成题。
- 全部完成后展示完成状态。

## 6. 完成状态

展示：

- 今日复习完成
- 返回首页
- 去薄弱巩固

# 7. 页面四：巩固

巩固 Tab 内包含两个入口：

1. 今日薄弱巩固
2. 专项训练

页面顶部可以使用两个大卡片入口。

## 今日薄弱巩固

采用和复习一致的题号导航模式。

展示：

- 5 题
- 来源标签：
  - 薄弱题型
  - 次薄弱题型
  - 随机挑战
- 查看答案
- 已掌握 / 仍需巩固
- 完成总结

交互：

- 一次只展示当前题。
- 题号导航横向滚动。
- 完成后展示总结卡片。

## 专项训练

入口：

- 学生选择三级题型开始训练。

流程：

```text
选择题型
↓
开始训练
↓
5题训练
↓
总结页
↓
未掌握题目可勾选加入错题库
```

页面要求：

- 题号导航
- 当前题详情
- 查看答案
- 已掌握 / 未掌握
- 独立训练总结页
- 未掌握题目支持勾选加入错题库

专项训练总结页：

- 总题数
- 已掌握数
- 未掌握数
- 未掌握题目列表
- 加入选中的错题库
- 再练一次
- 返回首页

- 分类状态筛选

- 题目预览
- 题型

操作：

- 查看答案
- 查看详情

建议：

- “查看答案”为主要操作。

## 4. 空状态

文案：

> 还没有错题，去录入第一道错题。

按钮：

- 录入错题

# 9. 页面六：答案解析页

答案解析页显示：

1. 题目
2. 题型
3. 答案
4. 解析
5. 返回来源页面

## 返回逻辑

必须支持 `returnUrl`。

来源规则：

- 从复习进入，返回复习。
- 从错题进入，返回错题。
- 从专项训练进入，返回专项训练。
- 如果没有 `returnUrl`，使用上一页返回。
- 再兜底返回错题页。

## 展示要求

- 题目、答案、解析均支持 LaTeX。
- 长公式允许横向滚动。
- 答案和解析分区清晰。
- 如果教师尚未补充，显示：

> 答案解析暂未补充，请等待老师更新。

# 14. LaTeX 展示策略

题目展示优先级：

1. `displayLatex`
2. `stem`

策略：

- 优先渲染 `displayLatex`。
- 渲染失败显示 `stem`。
- 长公式横向滚动。
- 题目、答案、解析都使用统一渲染器。

需要支持：

- `\blankbox`
- `\fourchoices`
- `\_\_\_`

## 自定义命令表现

### `\blankbox`

渲染为中文括号空格：

```text
（　　　）
```

### `\fourchoices`

渲染为四个选项：

```text
A. ...
B. ...
C. ...
D. ...
```

### `\_\_\_`

渲染为填空横线：

```text
__________
```
