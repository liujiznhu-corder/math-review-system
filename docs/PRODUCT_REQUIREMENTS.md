# 产品需求说明

## 产品名称

江苏专转本数学错题复盘系统

## 产品目标

帮助江苏专转本数学学生把错题按知识点和题型沉淀下来，并通过固定复习周期完成错题复盘。第一版目标是跑通“题型库维护 -> 错题录入 -> 自动推荐题型 -> 学生或教师确认 -> 生成复习任务 -> 今日复习”的闭环。

## 用户角色

### student

学生用户。由管理员或教师提前创建或邀请，不允许自行注册。

能力：

- 登录系统。
- 录入错题。
- 使用系统推荐题型。
- 选择已有题型并保存错题。
- 不确定时提交教师审核。
- 查看自己的错题库。
- 完成自己的今日复习任务。

限制：

- 不能新增、编辑、删除题型。
- 不能查看其他学生错题。
- 不能查看其他学生复习任务。

### teacher

教师用户。由管理员设置角色。

能力：

- 拥有学生侧能力。
- 管理题型库。
- 查看所有待审核错题。
- 为学生错题确认最终题型。

限制：

- 当前 MVP 不提供教师查看所有学生完整错题库的页面。
- 当前 MVP 不提供班级、课程、学生分组管理。

### admin

管理员用户。当前权限与 teacher 基本一致，用于系统初始化和最高权限管理。

能力：

- 拥有 teacher 能力。
- 创建或邀请用户后，可在 `profiles` 中调整角色。

## 核心业务对象

### 题型

题型必须来自数据库，不允许写死在代码中。

字段：

- 一级分类
- 二级分类
- 三级题型
- 关键词
- 例题
- 是否启用

用途：

- 为学生错题提供推荐分类。
- 为教师审核提供候选题型。
- 为错题库和复习页提供筛选和展示维度。

### 错题

学生手动录入错题。第一版不接 OCR，学生可以使用外部工具将题目转写为普通文本或 LaTeX 后复制进系统。

字段能力：

- 普通文本输入。
- LaTeX 输入。
- LaTeX 实时预览。
- 学生备注。
- 教师备注。
- 最终题型。
- 分类状态。

分类状态：

- `pending`：等待教师审核，暂不生成复习任务。
- `student_selected`：学生自己确认题型，可以生成复习任务。
- `teacher_confirmed`：教师确认题型，可以生成复习任务。

分类来源：

- `student`
- `teacher`
- `system`

当前 MVP 中，系统只推荐，不直接自动确认最终分类。

### 复习任务

当错题有确定题型后，系统生成复习任务。

初始复习周期：

- 第 1 天
- 第 3 天
- 第 7 天
- 第 14 天
- 第 30 天

任务状态：

- `pending`
- `completed`
- `skipped`

复习结果：

- `mastered`
- `not_mastered`
- `null`

未掌握后的补复习：

- 3 天后追加 `retry_day3`
- 7 天后追加 `retry_day7`

## 页面需求

### 登录页 `/login`

需求：

- 用户输入邮箱和密码登录。
- 登录成功进入应用。
- 不展示自由注册入口。

### 注册页 `/register`

需求：

- 不提供注册表单。
- 提示账号由管理员或教师创建。
- 可返回登录页。

### 题型库管理页 `/question-types`

访问权限：

- `teacher` / `admin` 可进入并修改。
- 学生导航不显示该入口。
- RLS 层面所有登录用户可读题型库。

功能：

- 查看题型列表。
- 新增题型。
- 编辑题型。
- 删除题型。
- 维护关键词。
- 维护例题。

### 错题录入页 `/mistakes/new`

访问权限：

- 登录用户可访问。

功能：

- 输入类型选择：普通文本 / LaTeX。
- 普通文本输入框。
- LaTeX 输入框。
- LaTeX 实时预览。
- 学生备注。
- 点击“智能推荐题型”后返回 top 3。
- 每个推荐结果展示：
  - 一级分类
  - 二级分类
  - 三级题型
  - 匹配分数
  - 匹配理由
- 学生可以选择推荐题型。
- 学生可以从已有题型中手动选择。
- 学生可以保存错题。
- 学生不确定时可以提交教师审核。

保存规则：

- 选择题型并保存：
  - `classification_status = student_selected`
  - `classified_by = student`
  - 自动生成复习任务。
- 提交教师审核：
  - `classification_status = pending`
  - `question_type_id = null`
  - 不生成复习任务。

### 错题库页 `/mistakes`

访问权限：

- 学生只能看自己的错题。

功能：

- 按题型筛选。
- 查看错题内容。
- LaTeX 错题渲染展示。
- 展示学生备注。
- 展示教师备注。
- 展示分类状态。
- 展示复习状态摘要：
  - 待复习任务数
  - 已完成复习次数
  - 最近一次复习结果

### 教师审核页 `/teacher/review-mistakes`

访问权限：

- 只有 `teacher` / `admin` 可访问。

功能：

- 显示所有 `classification_status = pending` 的错题。
- 展示错题题干。
- LaTeX 错题渲染展示。
- 展示学生备注。
- 展示系统推荐 top 3。
- 教师从已有题型中选择最终题型。
- 教师填写可选备注。
- 教师确认后更新原错题记录。
- 教师确认后为该学生生成复习任务。

### 今日复习页 `/reviews`

访问权限：

- 学生只能看自己的复习任务。

功能：

- 显示今日待复习数量。
- 显示今日已完成数量。
- 默认展示 `review_date <= 今天` 且 `status = pending` 的任务。
- 每条任务展示：
  - 错题题干
  - LaTeX 渲染内容
  - 所属题型
  - 复习轮次
  - 学生备注
- 提供“已掌握”按钮。
- 提供“未掌握”按钮。

操作规则：

- 已掌握：
  - 当前任务设为 `completed`
  - `result = mastered`
  - 写入 `completed_at`
- 未掌握：
  - 当前任务设为 `completed`
  - `result = not_mastered`
  - 写入 `completed_at`
  - 追加 3 天后和 7 天后的补复习任务。

## 分类需求

当前 MVP 不接 AI API，不接 OCR。

分类逻辑：

- 从 Supabase 读取启用的题型。
- 从 Supabase 读取题型例题。
- 对题干做文本归一化。
- 对 LaTeX 输入先去除 LaTeX 命令，再用于分类。
- 关键词命中加分。
- 例题相似度加分。
- 返回 top 3。

后续接 AI 时：

- 保留页面和调用方。
- 优先替换或扩展 `services/classifier.ts`。
- 仍然保留数据库题型库作为候选空间。

## 权限需求

- 不允许任何人前台自由注册。
- `question_types` 和 `question_type_examples`：
  - 所有登录用户可读。
  - 只有 `teacher` / `admin` 可写。
- `mistakes`：
  - 学生只能管理自己的错题。
  - `teacher` / `admin` 可读 pending 错题用于审核。
  - 教师审核写入通过服务端 service role client 完成。
- `review_tasks`：
  - 学生只能查看和更新自己的任务。
  - 创建学生复习任务可以通过 service role client 完成。

## 非目标

当前 MVP 不包含：

- OCR。
- AI API 分类。
- 图片上传。
- 班级管理。
- 作业管理。
- 支付。
- 复杂数据看板。
- 教师批量导入学生。
- 公开注册。
- 移动端原生 App。

## 成功标准

MVP 成功标准：

- 教师能维护题型库。
- 学生能录入普通文本和 LaTeX 错题。
- 系统能基于数据库题型推荐 top 3。
- 学生能自行确认题型或提交教师审核。
- 教师能确认 pending 错题题型。
- 确认题型后能生成复习任务。
- 学生能在 `/reviews` 完成复习任务。
- RLS 能隔离学生数据。
- `npm.cmd run lint` 和 `npm.cmd run build` 通过。
